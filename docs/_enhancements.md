# FIXES & ENHANCEMENTS 

Critical fixes, enhancements and configuration lessons for this OpenClaw deployment.

> **Updating this file:**
> 1. **Check first** — read the whole file before adding. If the fix relates to an
>    existing entry, add a sub-entry under it (e.g. `#12.16` after `#12.15`) instead
>    of appending a new top-level entry at the bottom.
> 2. Numbering is `#N.T` — `N` is the entry number in this file (sequential per
>    top-level section), `T` is the running total of all fixes (including minor ones
>    not documented here). Increment `T` for every new entry (top-level or sub-entry).
> 3. **Follow-up fixes get a part letter.** If you're patching, upgrading, or
>    revising something already documented, append a letter suffix instead of a new
>    number — e.g. `#21.35a`, `#21.35b`, `#21.35c`. This makes it clear the entries
>    are related but happened at different times. Only use a new number for genuinely
>    separate issues.
> 4. **Every entry must have a Date** (and Time where known, UTC or local with
>    timezone). This is mandatory — no exceptions. Readers need to see exactly when
>    each entry (and each part letter) was written.
>    Format: `9th Feb 2026` (ordinal day + 3-letter month + year).
> 5. Current total: **41**.
